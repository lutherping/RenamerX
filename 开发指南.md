批量文件重命名工具 (Batch File Renamer) 开发文档 v1.0
1. 项目概述
本项目是一个跨平台的桌面级批量文件/文件夹重命名工具。旨在通过直观的“积木式规则”和“实时预览”功能，降低用户批量处理文件的操作门槛，同时为高级用户提供正则表达式和 .bat 批处理脚本支持。

底层框架： Wails v2

后端语言： Go (Golang)

前端框架： Vue 3 (Composition API) + TypeScript

UI 组件库： Element Plus

2. 核心架构设计
应用采用标准的 Wails 前后端分离架构，通过 Wails 自动生成的 IPC (进程间通信) 桥接层进行数据交换。

前端 (Vue 3)： 负责状态管理（当前选中的文件列表、配置的重命名规则链）、UI 呈现、拦截用户输入并触发预览请求。

后端 (Go)： 负责繁重的文件系统 I/O 操作（读取目录、获取文件属性、执行物理重命名）、纯内存的字符串规则计算引擎、以及调用外部 .bat 脚本。

3. 界面布局定义 (UI Layout)
主界面采用经典的三分屏或左右分栏布局。

3.1 工作区 (Workspace - 顶部或左侧上方)
功能： 文件输入源。

组件： * 拖拽上传区域 (Dropzone)。

“添加文件” / “添加文件夹” 按钮。

清空列表按钮。

3.2 功能区 (Functional Area - 左侧侧边栏)
功能： 规则构建器 (Rule Builder)。用户在此处添加、删除、排序重命名规则。

UI 交互： * 提供一个“添加规则”的下拉菜单。

已经添加的规则以卡片 (Card) 形式上下排列，支持拖拽排序。

每张规则卡片内包含该规则所需的具体输入项（如：查找框、替换框）。

3.3 预览区 (Preview Area - 右侧主视图)
功能： 实时结果展示面板。

组件： * 一个数据表格 (Data Table)，包含以下列：

状态图标（等待中/成功/失败）

原文件名 (original_name)

所在路径 (path)

新文件名 (new_name) -> 核心关注点，需高亮

右下角的“执行重命名” (Execute) 主按钮。

4. 核心数据结构与接口 (API Design)
4.1 数据结构 (Go 结构体定义参考)
Go
// FileItem 代表一个待处理的文件实体
type FileItem struct {
	ID           string `json:"id"`            // 唯一标识 (UUID)
	OriginalName string `json:"originalName"`  // 原文件名 (带扩展名)
	Extension    string `json:"extension"`     // 扩展名 (.txt)
	Path         string `json:"path"`          // 绝对路径
	IsDir        bool   `json:"isDir"`         // 是否为文件夹
	NewName      string `json:"newName"`       // 预览/执行后的新文件名
	Status       string `json:"status"`        // 状态: "pending", "success", "error"
	ErrorMsg     string `json:"errorMsg"`      // 错误信息（如果有）
}

// RenameRule 代表一条重命名规则
type RenameRule struct {
	Type   string            `json:"type"`   // 规则类型: "replace", "prefix", "suffix", "regex", "script"
	Params map[string]string `json:"params"` // 规则参数 (例如: {"find": "旧文本", "replace": "新文本"})
}
4.2 Wails 暴露的后端方法 (Go Methods)
后端需要实现并在 app.go 中暴露以下核心方法供前端调用：

SelectFiles() []FileItem

描述： 呼出系统原生文件选择对话框，返回用户选中的文件列表信息。

SelectDirectory() []FileItem

描述： 呼出系统原生文件夹选择对话框，读取该目录下所有文件并返回。

PreviewNames(files []FileItem, rules []RenameRule) []FileItem

描述： 核心计算逻辑。 纯内存计算，不改动物理文件。遍历 files，将 rules 按顺序依次应用于 OriginalName，计算出 NewName 并返回更新后的列表。

注：当前端规则或文件列表发生变化时，防抖 (Debounce) 调用此方法。

ExecuteRename(files []FileItem) []FileItem

描述： 执行实际的物理重命名操作。调用 os.Rename。返回包含执行状态 (Status, ErrorMsg) 的最终结果列表供前端展示。

5. 外部脚本 (.bat) 支持规范
当 RenameRule.Type == "script" 时，触发 Go 的脚本引擎。

约定： 前端在 Params 中传入脚本的绝对路径 {"scriptPath": "C:\\my_script.bat"}。

执行方式： Go 通过 os/exec 调用 cmd.exe /c scriptPath "原文件名"。

安全性与性能： * 需要在 Go 层面设置执行超时时间（例如 2 秒），防止死循环脚本卡死整个应用。

由于大批量调用外部进程会导致性能开销巨大，可在前端提示用户：“使用外部脚本可能导致预览和执行速度变慢”。

6. 开发实施步骤 (Milestones)
阶段 1：脚手架搭建与 UI 骨架 (本周目标)

使用 wails init 初始化 Vue 3 模板。

引入 Element Plus。

画出左中右三大区域的静态界面，造一些假数据渲染表格。

阶段 2：Go 核心引擎与双向绑定

在 Go 中编写 PreviewNames 方法，实现基础的“添加前/后缀”、“替换”逻辑。

联调 Wails IPC，实现：前端修改输入框 -> 调用 Go 预览方法 -> 表格数据实时更新。

阶段 3：文件系统交互

实现 SelectFiles 和拖拽上传接口，获取真实本地文件。

实现 ExecuteRename 物理改名功能及错误回滚/跳过机制。

阶段 4：高级功能集成

引入正则表达式替换功能。

引入 .bat 外部脚本挂载与执行功能。

完善异常处理和用户通知（如文件占用提示）。
